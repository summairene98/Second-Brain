Questa cartella contiene i file di contesto a livello org. Sono documenti fondamentali che informano tutto il lavoro del vault.

## File

I file presenti dipendono dalla modalità (professional vs business) e dalle risposte date durante l'onboarding. Non tutti i file esistono in ogni vault.

| File | Scopo | Quando viene creato |
|---|---|---|
| `me.md` | Identità dell'operatore, stile di lavoro, loop non chiusi | Modalità Professional, sempre |
| `operator.md` | Ruolo dell'operatore, autorità decisionale, stato attuale, loop non chiusi | Modalità Business, sempre |
| `organization.md` | Struttura aziendale, missione, prodotti | Modalità Business, sempre |
| `business.md` | Contesto del business solo / aziendale | Modalità Professional, se applicabile |
| `services.md` | Servizi / prodotti attivi, baseline di fatturato, stack tecnologico | Se esistono linee di fatturato |
| `icp.md` | Profilo del cliente ideale | Se è stato nominato un cliente target |
| `pain-points.md` | Pain point dei clienti e come li affrontiamo | Se i pain point sono stati elencati |
| `infrastructure.md` | Stack di strumenti e integrazioni | Se gli strumenti sono stati elencati |
| `brand.md` | Voce, tono, messaggistica, identità visiva | Se il brand esiste |
| `team.md` | Elenco del team, accordi di lavoro, contatti esterni | Se esistono collaboratori |
| `strategy.md` | Obiettivi / OKR / priorità trimestrali | Se la strategia esiste |
| `stakeholders.md` | Fornitori, partner, investitori, advisor, clienti chiave | Modalità Business, se applicabile |

## Regole

- Preferenze, stile, abitudini dell'operatore → `me.md` (professional) o `operator.md` (business)
- Struttura org, informazioni aziendali, prodotti → `organization.md` (business) o `business.md` (professional)
- Linee di fatturato, servizi, prodotti → `services.md`
- ICP / profilo cliente → `icp.md`; pain point clienti → `pain-points.md`
- Modifiche allo stack di strumenti → `infrastructure.md`
- Brand / voce / tono / messaggistica / identità visiva → `brand.md`
- Elenco team, accordi di lavoro → `team.md`
- Strategia, OKR, obiettivi trimestrali → `strategy.md`
- Fornitori, partner, investitori, advisor → `stakeholders.md`
- Questi file sono fonti autorevoli. Non duplicare il loro contenuto altrove.
- Durante le revisioni settimanali, segnala gli obiettivi / OKR in `strategy.md` che non hanno nessun progetto attivo.
